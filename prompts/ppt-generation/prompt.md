# PPT 专业化改进指南

## 概述

本指南适用于将"简化版/概述型"PPT改进为"专业技术版"PPT的场景。核心思路是在保持精炼页数的同时，增加技术深度和信息密度。

## 适用场景

- 将面向业务人员的PPT改进为面向技术人员的版本
- 技术分享、论文汇报、架构评审等场景
- 需要展示技术细节但又不想做成40+页长篇的情况

## 改进维度

### 1. 结构重组

#### 问题-方案配对
将单页概述拆分为"问题页 + 方案页"配对：

```
❌ 简化版: 1页 "商品质量治理" (问题+方案混在一起)
✅ 专业版: 2页
   - 页1: "商品质量 - 问题与技术挑战" (痛点、攻击类型、技术指标要求)
   - 页2: "商品质量 - DTML解决方案" (算法流程、核心思想、结果数据)
```

#### 应用场景展开
将笼统的场景列表展开为结构化卡片：

```
❌ 简化版: "假货泛滥、虚假评论、违规广告"
✅ 专业版: 每个场景独立卡片，包含：
   - 具体问题描述
   - 技术挑战
   - 对应技术方案标签 (如 CLIP-ViT, BD-LLM, Graph分析)
```

### 2. 信息密度调整

#### 技术术语引入

| 简化版表述 | 专业版表述 |
|-----------|-----------|
| "同时看图+读文字" | "Cross-Attention建模跨模态关系" |
| "大模型的智慧，小模型的速度" | "BD-LLM框架：GPT-3.5(175B)→T5(3B)，60x压缩" |
| "像指纹一样匹配" | "全局+局部特征匹配，RANSAC几何验证" |

#### 数据指标突出

关键数字应该：
- 字号放大 (18-24pt)
- 带边框或背景色卡片
- 配合简短说明文字

```
┌─────────────┐
│    97%      │  ← 大字号、醒目颜色
│ 违规商品命中率│  ← 小字号说明
│"100件能抓97件"│  ← 可选的口语化解释
└─────────────┘
```

#### 论文引用格式

```
❌ 简化版: Multimodal Product Quality Prediction
✅ 专业版: WWW 2024 - Multimodal Product Quality Prediction
```

### 3. 视觉风格转换

#### 配色方案

| 风格 | 主色 | 强调色 | 适用场景 |
|------|------|--------|---------|
| 简化/业务 | 浅蓝 #E3F2FD | 蓝色 #2196F3 | 业务汇报 |
| 专业/技术 | 深蓝 #1A2332 | 科技蓝 #2563EB | 技术分享 |
| 学术风格 | 白色 #FFFFFF | 黑色 #1A1A1A | 论文演示 |

#### 去除卡通元素

```
❌ 使用 emoji 图标: 🛒 假货泛滥
✅ 使用结构化卡片:
   ┌────────────────┐
   │ 假货与盗图泛滥   │ ← 标题 (红色醒目)
   │ 传统哈希算法失效  │ ← 描述 (灰色正文)
   └────────────────┘
```

### 4. 布局原则

#### 双栏布局

适合"内容 + 数据"并列展示：

```
┌──────────────────────────────────────┐
│            Header (深色背景)           │
├────────────────────┬─────────────────┤
│                    │    ┌───────┐    │
│   技术方案详解       │    │ 97%  │    │
│   - 步骤1           │    │ 命中率 │    │
│   - 步骤2           │    └───────┘    │
│   - 步骤3           │    ┌───────┐    │
│                    │    │1210万 │    │
│   核心思想框        │    │ 挽回   │    │
│                    │    └───────┘    │
└────────────────────┴─────────────────┘
     左栏 (60%)            右栏 (40%)
```

#### 四象限/四支柱布局

适合展示并列的分类体系：

```
┌─────────┬─────────┬─────────┬─────────┐
│ Safety  │Integrity│Generation│Knowledge│
│ 安全    │ 诚信    │ 生成     │ 知识    │
├─────────┼─────────┼─────────┼─────────┤
│ BD-LLM  │ I-SEE   │ RAG     │ COSMO   │
│ TaeBench│ DTML    │ 文案生成 │ 意图识别 │
└─────────┴─────────┴─────────┴─────────┘
```

## 改进流程

### Step 1: 分析原始PPT

```bash
# 提取文本内容
python -m markitdown original.pptx > content.md

# 生成缩略图
python scripts/thumbnail.py original.pptx thumbnails --cols 5
```

评估要点：
- 当前信息密度
- 技术术语使用程度
- 目标受众匹配度

### Step 2: 确定改进策略

根据目标受众选择改进方向：

| 目标受众 | 改进重点 |
|---------|---------|
| 技术工程师 | 算法细节、代码示例、架构图 |
| 技术管理层 | 技术选型理由、ROI数据、风险分析 |
| 研究人员 | 论文引用、实验数据、创新点 |

### Step 3: 内容重组

1. 识别可拆分的概述页
2. 补充技术细节 (从原始详细版本或相关资料)
3. 添加数据指标卡片
4. 规范化论文引用

### Step 4: 视觉优化

1. 统一配色方案
2. 去除卡通元素
3. 突出关键数据
4. 保持布局一致性

### Step 5: 验证

```bash
# 生成新版本缩略图对比
python scripts/thumbnail.py improved.pptx thumbnails_new --cols 4
```

检查清单：
- [ ] 信息密度是否提升
- [ ] 技术术语是否准确
- [ ] 数据指标是否突出
- [ ] 视觉风格是否统一
- [ ] 页数是否合理 (建议15-25页)

### 封面页

```
❌ 简化版:
   Amazon AI 内容审核实践
   用 AI 守护电商信任
   [emoji装饰]

✅ 专业版:
   Amazon AI 内容审核技术实践
   多模态融合 · LLM蒸馏 · 对抗鲁棒性
   [Applied Science 标签]
```

### 技术方案页

```
❌ 简化版:
   💡 方案：多模态检测 + 图像指纹
   1. 提取图像深度特征
   2. 图文联合理解
   3. 全局+局部匹配
   [大数字] 97% 命中率

✅ 专业版:
   DTML: Deep Metric Learning + Global-Local Matching

   ┌─[1]─────────────────────────────┐
   │ 数据增强 (Data Augmentation)     │
   │ 模拟真实攻击：裁剪、旋转、滤镜    │
   │ [Contrastive Learning] [Hard Negative Mining] │
   └─────────────────────────────────┘

   ┌─[2]─────────────────────────────┐
   │ Embedding 模型训练               │
   │ DTML在嵌入空间中：复制品拉近，    │
   │ 外观相似但非复制品推远            │
   │ [Triplet Loss] [Angular Loss]   │
   └─────────────────────────────────┘

   核心思想框:
   "传统方法无法区分'长得像'和'盗用'。
    DTML学习语义级相似度，结合局部特征匹配消除误报。"

   ┌────────┐  ┌────────┐
   │  97%   │  │ 1210万 │
   │ 命中率  │  │挽回误封 │
   └────────┘  └────────┘

   参考论文:
   CVPR 2023 - Global-Local Image Perceptual Score
   KDD 2023 - Unified Image-Text Embedding
```

## 注意事项

1. **不要过度技术化**: 保持可读性，避免堆砌术语
2. **保持逻辑连贯**: 技术细节应服务于整体叙事
3. **数据要准确**: 引用的论文和数据指标必须准确
4. **控制页数**: 专业版不应比简化版多太多页 (建议增加20-30%)
